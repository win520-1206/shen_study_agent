## 三、当前系统最像"原型"的地方

### 1. 内容生成仍然是"模板拼接"

当前 ContentGeneratorAgent 虽然能根据画像选择不同模板，但本质上还是在拼字符串。讲义的正文内容来自知识库的固定切片，无法动态重组。

**升级方向：** 接入真实大模型后，让 LLM 根据画像和知识库内容实时生成个性化讲义。

### 2. 画像识别是关键词匹配

ProfileAgent 用 `any(k in message for k in [...])` 做判断，无法理解否定句、转折句和复杂意图。

**升级方向：** LLMProfileAgent 已实现，接入真实 API 后替换即可。

### 3. 缺少真实大模型生成

MockLLMService 返回固定模板，不是真正的个性化生成。

**升级方向：** 接入科大讯飞 Spark API，用真实大模型替代 Mock。

### 4. 前端没有流式输出

所有内容是请求完成后一次性展示，没有"打字机效果"。

**升级方向：** 后端支持 SSE 流式输出，前端逐字渲染。

### 5. 缺少智能答疑和学习评估

当前只能"生成方案"，不能"回答问题"和"评估效果"。

**升级方向：** 第四周实现加分功能。

## 四、接真实大模型从哪入手

### 第一步：获取科大讯飞 API 凭证

1. 注册科大讯飞开放平台账号
2. 创建星火大模型应用
3. 获取 APP_ID、API_KEY、API_SECRET

### 第二步：实现 XfyunLLMService

当前 `llm_service.py` 中的 `XfyunLLMService` 是空壳（fallback 到 Mock），需要实现真实 API 调用。参考科大讯飞 WebSocket 或 HTTP 接口文档。

### 第三步：修改配置

```python
# config.py
USE_LLM = True
LLM_PROVIDER = "xfyun"
XFYUN_APP_ID = "真实 APP_ID"
XFYUN_API_KEY = "真实 API_KEY"
XFYUN_API_SECRET = "真实 API_SECRET"
```

### 第四步：验证

- 前端零改动
- 数据库零改动
- 同样的 3 种学生预设，观察 LLM 生成的内容是否比规则式更自然

**预计工作量：** 1-2 天（主要时间在 API 调试和 prompt 优化）

## 五、当前最值得展示给评委的是什么

按优先级排序：

### 1. 多智能体协作管线（最值得展示）

6 个智能体分工协作的水平流程图，是视觉上最有冲击力的展示。评委一看就知道"这不是普通问答机器人"。

### 2. 3 种学生预设的个性化差异

一键切换零基础/项目实战/考试冲刺，展示画像、资源、路径全部不同。这是最直接证明"个性化"的方式。

### 3. 5 类资源的差异化生成

讲义、题目、代码案例、思维导图、学习路径——不同类型 + 不同内容 + 不同风格。展示资源的丰富度和个性化深度。

### 4. 知识库 RAG 架构

每条资源都标注了来源（source_refs），展示"不是凭空生成，而是基于知识库"。这是和普通 ChatGPT 的核心区别。

### 5. 科技感深色主题前端

整体视觉效果已经具备比赛演示水准，不用再花时间美化。
