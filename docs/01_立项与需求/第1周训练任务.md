# 第 1 周训练任务

这份文档不是普通排期表，而是你边做边学的带练脚本。  
目标是：在 7 天内真正吃透赛题、理解项目结构、能独立讲清楚项目要做什么，并亲手跑通第一条后端主链路。

## 第 1 天：把赛题翻译成自己的话

### 要做什么

1. 阅读 `赛题需求清单.md`
2. 阅读 `功能优先级表.md`
3. 自己写一段 200-300 字的“我理解的赛题”

### 为什么做

很多同学一上来就写代码，最后做出来的东西和赛题不匹配。  
先把赛题翻译成自己的语言，才能知道哪些功能是必须做，哪些只是锦上添花。

### 你会学到什么

- 什么是需求分析
- 什么是“赛题要求”和“自己想做的功能”之间的区别
- 什么是 MVP

### 完成标准

你能脱稿说清楚下面 4 件事：

1. 为什么系统一定要有“多智能体”
2. 为什么一定要有 6 维以上画像
3. 为什么一定要有 5 类资源
4. 为什么必须自建一门完整课程知识库

## 第 2 天：理解项目目录和软著留痕逻辑

### 要做什么

1. 阅读项目根目录 `README.md`
2. 打开 `docs`、`evidence`、`src`、`data` 看每个目录的用途
3. 用自己的话记录每个目录是干什么的

### 为什么做

比赛项目不是只交代码。  
你必须从一开始就养成“代码、文档、证据一起做”的习惯，这样后面写说明书、做答辩、准备软著都会轻松很多。

### 你会学到什么

- 什么是工程化目录
- 什么是开发留痕
- 为什么软著不等于最后导出一份代码

### 完成标准

你能解释：

- `docs` 为什么不能等最后再补
- `evidence` 为什么对比赛和软著都有价值
- `data/course_kb` 为什么是这个项目的核心资产之一

## 第 3 天：理解课程知识库怎么组织

### 要做什么

1. 阅读 `data/course_kb/metadata.json`
2. 阅读至少 2 个模块文件，建议先看 `ml03_linear_regression.md` 和 `ml08_evaluation.md`
3. 阅读 `question_bank.json` 和 `coding_cases.json`
4. 画出“模块 -> 知识点 -> 题目 -> 代码案例”的关系

### 为什么做

这个项目不是让大模型凭空回答，而是让它基于课程资料生成个性化资源。  
你要先明白知识库结构，后面才能理解检索、资源生成和防幻觉。

### 你会学到什么

- 什么是知识库元数据
- 什么是结构化学习资源
- 为什么题库和代码案例也属于知识库的一部分

### 完成标准

你能解释：

1. `metadata.json` 解决了什么问题
2. 为什么 Markdown 适合存课程内容
3. 为什么知识库要同时有讲义、题目和代码案例

## 第 4 天：理解后端主流程

### 要做什么

1. 阅读 `src/backend/app/main.py`
2. 阅读 `src/backend/app/routers/students.py`
3. 阅读 `src/backend/app/services/orchestrator.py`
4. 画出“用户输入后，系统内部发生了什么”

### 为什么做

你后面不管是自己改功能，还是讲给评委听，都必须真正明白系统主链路。

### 你会学到什么

- 什么是 API 路由
- 什么是服务层
- 什么是编排器
- 什么是一次完整请求的处理流程

### 完成标准

你能按顺序讲出：

1. 用户消息进入哪个接口
2. 哪几个智能体依次参与
3. 哪些结果会写入数据库
4. Dashboard 是怎么读回这些结果的

## 第 5 天：理解多智能体设计

### 要做什么

1. 阅读 `src/backend/app/services/agents.py`
2. 给每个智能体写一句“它负责什么”
3. 给每个智能体写一句“如果没有它会怎样”

### 为什么做

赛题特别强调“多智能体”，所以你不能只是把几个函数名字改成 agent。  
你要能清楚证明：每个角色都有职责，协作后才能形成完整学习服务。

### 你会学到什么

- 什么是角色拆分
- 什么是可解释的智能体系统
- 为什么“一个大模型做完所有事”不利于比赛展示

### 完成标准

你能把 6 个智能体分别讲给别人听，而且不混淆它们的职责。

## 第 6 天：亲手跑通接口

### 要做什么

1. 执行 `pip install -r requirements.txt`
2. 执行 `uvicorn src.backend.app.main:app --reload`
3. 打开 `http://127.0.0.1:8000/docs`
4. 依次测试：
   - `POST /api/v1/students`
   - `POST /api/v1/chat/profile-build`
   - `POST /api/v1/assessment/submit`
   - `GET /api/v1/student/{student_id}/dashboard`

### 为什么做

只有你自己点过、看过返回值、追过一次完整数据链路，项目才真的开始变成“你会的东西”。

### 你会学到什么

- 什么是 Swagger
- 什么是请求体和响应体
- 什么是接口联调的最小方法

### 完成标准

你能独立创建一个学生，并让系统返回一份画像、资源和学习路径。

## 第 7 天：写自己的第 1 周总结

### 要做什么

写一份不低于 500 字的周总结，至少包含：

1. 我现在理解的赛题是什么
2. 我现在理解的系统结构是什么
3. 我目前最懂的部分是什么
4. 我目前最不懂的部分是什么
5. 下周我准备优先补什么

### 为什么做

总结不是形式主义，它会逼你把“看过”变成“会讲”，把“模糊感觉”变成“清晰问题”。

### 你会学到什么

- 如何做项目复盘
- 如何定位自己的真实薄弱点
- 如何把学习过程转化成比赛表达材料

### 完成标准

你能基于这一周的内容，对别人讲清楚 LearnMate-AI 的基本架构和主流程。

## 第 1 周验收标准

如果这一周结束时，你能做到以下几点，就说明节奏是对的：

1. 能完整复述赛题要求
2. 能说出项目目录为什么这样设计
3. 能解释知识库的结构和作用
4. 能说明 6 个智能体的职责
5. 能亲手跑通后端主链路
6. 能写出自己的学习总结和问题清单

## 第 1 周不要做的事

1. 不要一开始就追求复杂大模型接入
2. 不要急着做花哨 UI 而忽略主流程
3. 不要跳过文档和证据留存
4. 不要在没理解结构前盲目加功能

## 给你的提醒

这一周最重要的不是“做了多少功能”，而是建立正确的项目理解框架。  
只要你真正弄懂了赛题、知识库、后端主链路和多智能体分工，后面的功能扩展就会快很多。
