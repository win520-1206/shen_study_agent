# 项目简介

## 项目名称

LearnMate-AI：基于大模型的个性化学习多智能体系统

## 一句话介绍

学生只需说一句话描述自己的学习情况，6 个智能体就会自动协作，为他生成一套只适合他的学习方案——包括讲义、练习题、代码案例、思维导图和学习路径。

系统现已支持云端访问、知识图谱盲区诊断、苏格拉底式追问、交互式自动批改、学习效果追踪和学生持续跟踪。

## 目标用户

- 高校人工智能、计算机、电子信息相关专业学生
- 需要围绕某门课程做个性化自主学习的学习者

## 核心痛点

1. 学生基础差异大，但现有学习工具做不到真正个性化
2. 学习资源（讲义、习题、代码案例）分散，缺乏体系
3. 学生不清楚自己的薄弱环节，盲目学习效率低
4. 纯大模型问答缺乏课程知识库支撑，内容可靠性不足
5. 传统练习没有即时反馈和批改，学习闭环断裂

## 系统能力

### 多智能体协作架构

| 智能体 | 职责 | 类比 |
|--------|------|------|
| 画像智能体 | 从对话中提取 8 维学习画像 | 学情调查员 |
| 诊断智能体 | 分析薄弱环节，推荐优先模块 | 学情分析师 |
| 资源规划智能体 | 决定资源难度、风格和类型 | 课程顾问 |
| 内容生成智能体 | 生成 5 类个性化学习资源 | 备课老师 |
| 路径规划智能体 | 设计分阶段学习路径 | 学习教练 |
| 审查智能体 | 校验内容完整性和来源可追溯性 | 质检员 |
| 自动批改智能体 | 对练习题进行 LLM 评分和反馈 | AI 助教 |

系统会保留每个智能体的输入摘要、输出摘要、决策原因和对最终结果的影响，便于展示"为什么这样推荐"。

### 个性化学习资源（5 类）

1. 知识讲解文档
2. 练习题（支持选择题 + 简答题混合题型）
3. 代码实操案例
4. 思维导图（Mermaid）
5. 学习路径清单

### 学生画像维度（8 维）

1. 专业背景
2. 学习目标（期末提分/入门掌握/项目实战/考研准备/竞赛准备）
3. 基础水平（零基础/基础一般/有编程基础/学过部分算法）
4. 薄弱知识点
5. 学习风格（看讲解/做题/代码实战/图示化总结）
6. 每周投入时间
7. 练习偏好（选择题/简答题/编程题/混合题型）
8. 期望结果

### 知识图谱与盲区诊断

系统内置机器学习课程知识图谱（8 个模块、依赖关系边），前端使用 ECharts 可视化渲染。当诊断出学生某知识点薄弱时，图谱自动高亮该节点及其上游依赖，提示"建议先巩固前置知识"。将个性化从简单的"推资源"提升到学科逻辑推演层面。

### 苏格拉底式追问

智能答疑支持"苏格拉底模式"开关。开启后，AI 不仅回答问题，还会在回答末尾主动抛出一个引导性问题，促使学生深入思考。例如：学生问"什么是过拟合？"，AI 回答后会追问"你能想出一种正则化方法来缓解过拟合吗？"

### 交互式练习题与自动批改

练习题页面支持选择题（A/B/C/D 即时判对错）和简答题（LLM 自动评分 0-100 分 + 反馈 + 要点命中/遗漏）。答题完成后自动提交评估成绩到后端，无需手动操作。系统根据 `exercise_preference` 自动调整题型比例。

### 学生持续跟踪

通过 localStorage 持久化学生身份，刷新页面自动恢复。侧边栏学生选择器支持切换已有学生或新建学生，同一学生多次"生成"不会重复创建记录，评估历史跨次累积。

## 技术架构

- 后端：FastAPI + SQLAlchemy + SQLite
- 前端：Vue 3 + Vite + Element Plus + vue-router + ECharts
- 知识库：8 个机器学习基础模块，含讲义、32 道题、10 个代码案例 + 知识依赖图谱
- 大模型接入：支持 Mock 模式和 DeepSeek API（一行配置切换）
- 部署形态：已部署至阿里云 ECS，支持公网访问与演示

## 创新点

1. **多智能体协作式个性化生成**：6+1 个智能体分工协作，而非单一 AI 回答
2. **知识库驱动的 RAG 架构**：生成内容基于课程知识库，减少幻觉，可追溯
3. **知识图谱盲区诊断**：基于学科依赖关系的智能薄弱点分析与前置知识推荐
4. **苏格拉底式追问引导**：AI 主动引导学生深度思考，而非被动回答
5. **交互式自动批改**：选择题即时判对错 + 简答题 LLM 评分，自动记录成绩
6. **配置驱动的大模型无缝切换**：稳定接口设计，一行配置切换模式
7. **学习闭环与推荐依据可解释**：系统会根据评估历史调整推荐重点，并明确展示推荐依据与成长变化
8. **学生持续跟踪与多画像管理**：支持多名学生独立跟踪，侧边栏快速切换

## 与传统方式的区别

| 对比维度 | 传统方式 | LearnMate-AI |
|----------|----------|-------------|
| 学情了解 | 问卷/考试 | 自然语言对话，实时画像 |
| 资源获取 | 自行拼凑 | 系统自动生成，一站式 |
| 个性化程度 | 全班一套教案 | 每人一套专属方案 |
| 内容质量 | 取决于来源 | 知识库 + 审查校验 |
| 薄弱诊断 | 考试后才知道 | 知识图谱实时分析依赖关系 |
| 练习反馈 | 等老师批改 | LLM 即时评分 + 要点分析 |
| 学习引导 | 被动接受 | 苏格拉底式主动追问 |
| 跟踪持续性 | 每次独立 | localStorage 持久化，跨次累积 |
