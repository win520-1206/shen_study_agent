# 项目简介

## 项目名称

LearnMate-AI：基于大模型的个性化学习多智能体系统

## 一句话介绍

学生只需说一句话描述自己的学习情况，6 个智能体就会自动协作，为他生成一套只适合他的学习方案——包括讲义、练习题、代码案例、思维导图和学习路径。

## 目标用户

- 高校人工智能、计算机、电子信息相关专业学生
- 需要围绕某门课程做个性化自主学习的学习者

## 核心痛点

1. 学生基础差异大，但现有学习工具做不到真正个性化
2. 学习资源（讲义、习题、代码案例）分散，缺乏体系
3. 学生不清楚自己的薄弱环节，盲目学习效率低
4. 纯大模型问答缺乏课程知识库支撑，内容可靠性不足

## 系统能力

### 多智能体协作架构

| 智能体 | 职责 |
|--------|------|
| 画像智能体 | 从对话中提取 8 维学习画像 |
| 诊断智能体 | 分析薄弱环节，推荐优先模块 |
| 资源规划智能体 | 决定资源难度、风格和类型 |
| 内容生成智能体 | 生成 5 类个性化学习资源 |
| 路径规划智能体 | 设计分阶段学习路径 |
| 审查智能体 | 校验内容完整性和来源可追溯性 |

### 个性化学习资源（5 类）

1. 知识讲解文档
2. 练习题
3. 代码实操案例
4. 思维导图（Mermaid）
5. 学习路径清单

### 学生画像维度（8 维）

1. 专业背景
2. 学习目标（期末提分/入门掌握/项目实战/考研准备/竞赛准备）
3. 基础水平（零基础/基础一般/有编程基础/学过部分算法）
4. 薄弱知识点
5. 学习风格（看讲解/做题/代码实战/图示化总结）
6. 每周投入时间
7. 练习偏好（选择题/简答题/编程题/混合题型）
8. 期望结果

## 技术架构

- 后端：FastAPI + SQLAlchemy + SQLite
- 前端：Vue 3 + Vite + Element Plus
- 知识库：8 个机器学习基础模块，含讲义、32 道题、10 个代码案例
- 大模型接入：支持 Mock 模式和科大讯飞星火 API（一行配置切换）

## 创新点

1. **多智能体协作式个性化生成**：6 个智能体分工协作，而非单一 AI 回答
2. **知识库驱动的 RAG 架构**：生成内容基于课程知识库，减少幻觉，可追溯
3. **配置驱动的大模型无缝切换**：稳定接口设计，一行配置切换模式

## 与传统方式的区别

| 对比维度 | 传统方式 | LearnMate-AI |
|----------|----------|-------------|
| 学情了解 | 问卷/考试 | 自然语言对话，实时画像 |
| 资源获取 | 自行拼凑 | 系统自动生成，一站式 |
| 个性化程度 | 全班一套教案 | 每人一套专属方案 |
| 内容质量 | 取决于来源 | 知识库 + 审查校验 |
