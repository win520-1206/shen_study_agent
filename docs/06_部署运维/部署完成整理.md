## 3. 后续更新代码的标准流程

### 推荐方式：Git 强制同步

```bash
# 本地提交
cd C:\shen_study_agent
git add .
git commit -m "描述修改内容"
git push

# 服务器更新
ssh root@121.199.26.183
cd /opt/shen_study_agent
git fetch origin
git reset --hard origin/main
```

> **为什么不用 git pull？** 服务器上的 `runtime/learnmate.db` 和 `config.py` 等文件会导致 merge 冲突，`git reset --hard` 可以避免这个问题。

### 如果改了前端代码

```bash
cd /opt/shen_study_agent/src/frontend
node node_modules/vite/bin/vite.js build
```

> **为什么不用 npm run build？** 服务器上 npm 的 vite 可执行文件有权限问题，直接用 node 调用即可。

### 重启服务

```bash
systemctl restart learnmate
```

### 如果改了后端依赖

```bash
cd /opt/shen_study_agent
source venv/bin/activate
pip install -r requirements.txt
systemctl restart learnmate
```

## 4. 常用维护命令

```bash

systemctl status learnmate

# 重启服务
systemctl restart learnmate

# 查看实时日志
journalctl -u learnmate -f

# 查看最近日志
journalctl -u learnmate -n 100

# 检查端口
ss -ltnp | grep 8000

# 服务器自检
curl http://127.0.0.1:8000
```

## 5. 注意事项

1. `config.py` 不要上传到 GitHub，服务器上单独维护。
2. 每次 `git reset --hard` 后必须重新构建前端。
3. Node.js 18 是通过二进制安装到 `/usr/local/bin/node`，不要用 apt 安装。
4. 如果 `npm` 命令找不到，用 `node node_modules/vite/bin/vite.js build` 代替。
