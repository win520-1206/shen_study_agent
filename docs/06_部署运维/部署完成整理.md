## 3. 为什么之前会启动失败

之前服务器启动失败的核心原因是：

1. 服务器默认 Python 版本是 `3.10`。
2. 项目代码中使用了 `datetime.UTC`。
3. `datetime.UTC` 需要较新的 Python 版本支持。
4. 所以在 Python 3.10 下启动时报错：

```text
ImportError: cannot import name 'UTC' from 'datetime'
```

后来已经通过安装 Python 3.13 并重建虚拟环境解决。

## 4. 现在项目是怎么启动的

你现在的项目有两种启动方式。

### 4.1 临时手动启动

适合调试时临时使用：

```bash
cd /opt/shen_study_agent
source venv/bin/activate
python -m uvicorn src.backend.app.main:app --host 0.0.0.0 --port 8000
```

特点：

- 当前终端窗口不能关
- 关掉终端后服务就停止

### 4.2 正式服务启动

适合长期运行，推荐使用：

```bash
systemctl restart learnmate
```

特点：

- 后台运行
- 不需要一直开着 SSH 窗口
- 可以开机自启
- 更适合比赛演示和长期访问

## 9. 配置文件和 API Key 注意事项

你现在一定要注意下面这件事：

```text
src/backend/app/config.py 不能上传到 GitHub
```

原因是这个文件里可能有：

1. API Key
2. 模型地址
3. 真实运行配置

仓库里应该保留的是模板文件：

```text
src/backend/app/config.example.py
```

服务器上的真实配置文件位置：

```text
/opt/shen_study_agent/src/backend/app/config.py
```

如果 API Key 有泄露风险，正确处理方式是：

1. 去平台后台删除旧 Key
2. 重新生成新 Key
3. 修改服务器上的 `config.py`
4. 执行 `systemctl restart learnmate`
