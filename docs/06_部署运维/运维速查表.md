# LearnMate-AI 运维速查表

> 适用场景：项目已经部署到阿里云服务器，后续需要更新代码、重启服务、查看日志、备份配置时使用。

---

## 1. 最常用的更新流程

### 本地提交代码

```bash
cd C:\shen_study_agent
git add .
git commit -m "写清楚这次改了什么"
git push
```

### 服务器拉取并重启

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git pull
systemctl restart learnmate
```

---

## 2. 不同改动对应的处理方式

### 情况 1：只改了后端 Python 代码

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git pull
systemctl restart learnmate
```

### 情况 2：改了前端 Vue 页面

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git pull
cd src/frontend
npm install
npm run build
cd /opt/shen_study_agent
systemctl restart learnmate
```

> **重要提醒**：`git pull` 或 `git reset --hard` 会覆盖 `src/backend/static/` 下的构建产物，
> 即使只改了后端代码，如果前端静态文件被重置了也必须重新构建前端。
> 养成习惯：每次 `git pull` / `git reset` 之后都跑一次前端构建。
>
> 如果服务器上 `npm run build` 报 `vite: Permission denied`，用以下命令代替：
> ```bash
> node node_modules/vite/bin/vite.js build
> ```
> 如果报 `npx: not found`，也用上面的命令。
### 情况 3：改了 `requirements.txt`

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git pull
source venv/bin/activate
pip install -r requirements.txt
systemctl restart learnmate
```

### 情况 4：改了 `config.py`

```bash
ssh root@121.199.26.183
nano /opt/shen_study_agent/src/backend/app/config.py
systemctl restart learnmate
```

## 3. 服务管理命令

### 查看服务状态

```bash
systemctl status learnmate
```

### 重启服务

```bash
systemctl restart learnmate
```

### 停止服务

```bash
systemctl stop learnmate
```

### 启动服务

```bash
systemctl start learnmate
```

### 查看是否开机自启

```bash
systemctl is-enabled learnmate
```

正常应返回：

```text
enabled
```

## 7. 安全提醒

### `config.py` 不要上传到 GitHub

仓库中应只保留：

```text
src/backend/app/config.example.py
```

真实密钥只放在服务器里的：

```text
/opt/shen_study_agent/src/backend/app/config.py
```

### 如果 API Key 泄露

1. 去对应平台后台删除旧 key
2. 生成新 key
3. 修改服务器 `config.py`
4. 重启服务

```bash
systemctl restart learnmate
```

## 9. 常见问题

### 浏览器打不开

先执行：

```bash
systemctl status learnmate
ss -ltnp | grep 8000
curl http://127.0.0.1:8000
```

### 前端页面空白

```bash
cd /opt/shen_study_agent/src/frontend
npm install
node node_modules/vite/bin/vite.js build
systemctl restart learnmate
```

### git pull / git reset 后页面消失

这是最常见的问题。`git` 操作会重置 `src/backend/static/` 目录，必须重新构建前端：

```bash
cd /opt/shen_study_agent/src/frontend
node node_modules/vite/bin/vite.js build
systemctl restart learnmate
```

### 服务启动失败

```bash
journalctl -u learnmate -n 100
```
