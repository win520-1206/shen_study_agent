## 1. 最常用的更新流程

### 本地提交代码

```bash
cd C:\shen_study_agent
git add .
git commit -m "写清楚这次改了什么"
git push
```

### 服务器拉取并重启

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git fetch origin
git reset --hard origin/main
systemctl restart learnmate
```

> **为什么不用 git pull？** 服务器上的 `runtime/learnmate.db` 等文件会导致 merge 冲突。

## 2. 不同改动对应的处理方式

### 情况 1：只改了后端 Python 代码

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git fetch origin && git reset --hard origin/main
systemctl restart learnmate
```

### 情况 2：改了前端 Vue 页面

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git fetch origin && git reset --hard origin/main
cd src/frontend && node node_modules/vite/bin/vite.js build
cd /opt/shen_study_agent
systemctl restart learnmate
```

> **重要**：每次 `git reset --hard` 后 `src/backend/static/` 下的构建产物会被覆盖，必须重新构建前端。
>
> 如果 `npm run build` 报 `vite: Permission denied`，用：
> ```bash
> node node_modules/vite/bin/vite.js build
> ```

### 情况 3：改了 `requirements.txt`

```bash
ssh root@121.199.26.183
cd /opt/shen_study_agent
git fetch origin && git reset --hard origin/main
source venv/bin/activate
pip install -r requirements.txt
systemctl restart learnmate
```

### 情况 4：改了 `config.py`

```bash
ssh root@121.199.26.183
nano /opt/shen_study_agent/src/backend/app/config.py
systemctl restart learnmate
```

## 9. 常见问题

### Q: 网页打不开？

1. 检查服务是否运行：`systemctl status learnmate`
2. 检查端口是否监听：`ss -ltnp | grep 8000`
3. 检查安全组是否开放 8000 端口
4. 检查前端是否构建：`ls src/backend/static/index.html`

### Q: 页面内容没有更新？

1. 确认已构建前端：`cd src/frontend && node node_modules/vite/bin/vite.js build`
2. 确认已重启服务：`systemctl restart learnmate`
3. 清除浏览器缓存（Ctrl+Shift+R）

### Q: 500 Internal Server Error？

1. 查看日志：`journalctl -u learnmate -n 50`
2. 常见原因：LLM API 调用失败、JSON 解析错误
3. 临时方案：将 `config.py` 中 `USE_LLM` 设为 `False` 使用 Mock 模式

### Q: Network Error 前端报错？

1. 确认后端服务正常：`curl http://127.0.0.1:8000/api/v1/students`
2. 检查前端 API 地址配置是否指向正确的后端地址
3. 检查浏览器控制台具体报错信息
