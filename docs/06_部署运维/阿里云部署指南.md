# 阿里云部署指南

> 更新日期：2026-06-04  
> 适用系统：Ubuntu 22.04 LTS  
> 推荐 Python：3.13  
> 推荐 Node.js：18（二进制安装）

---

## 一、购买阿里云服务器

### 1.1 推荐配置

| 项目 | 推荐值 |
|------|--------|
| 实例类型 | ECS 经济型 (ecs.t6-c1m1.large) |
| CPU/内存 | 1核 2G |
| 操作系统 | Ubuntu 22.04 LTS 64位 |
| 存储 | 40G 高效云盘 |
| 带宽 | 1-3 Mbps（按量付费） |
| 地域 | 华东1-杭州 |

### 1.2 安全组配置

| 协议 | 端口 | 来源 |
|------|------|------|
| TCP | 8000 | 0.0.0.0/0 |

## 四、安装 Node.js 18（二进制方式）

> **重要：不要用 apt 安装 Node.js**，系统自带的版本太旧（v12），且 apt 安装的 NodeSource 版本会与 `libnode72` 冲突。

```bash
cd /tmp
curl -fsSL https://nodejs.org/dist/v18.20.4/node-v18.20.4-linux-x64.tar.xz -o node18.tar.xz
tar -xf node18.tar.xz
cp -f node-v18.20.4-linux-x64/bin/node /usr/local/bin/node
cp -rf node-v18.20.4-linux-x64/lib/node_modules /usr/local/lib/node_modules
ln -sf /usr/local/lib/node_modules/npm/bin/npm-cli.js /usr/local/bin/npm
node --version    # 应显示 v18.20.4
npm --version     # 应显示 10.x
```

## 六、服务器配置

### 6.1 创建配置文件

```bash
cd /opt/shen_study_agent
cp src/backend/app/config.example.py src/backend/app/config.py
nano src/backend/app/config.py
```

修改以下配置：

```python
USE_LLM = True
LLM_PROVIDER = "generic"
LLM_BASE_URL = "https://api.deepseek.com"
LLM_API_KEY_V2 = "sk-你的真实密钥"
LLM_MODEL = "deepseek-chat"
```

### 6.2 创建虚拟环境并安装依赖

```bash
cd /opt/shen_study_agent
python3.13 -m venv venv
source venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### 6.3 构建前端

```bash
cd /opt/shen_study_agent/src/frontend
npm install
node node_modules/vite/bin/vite.js build
cd /opt/shen_study_agent
```

> **注意**：如果 `npm run build` 报 `vite: Permission denied` 或 `npx: not found`，使用 `node node_modules/vite/bin/vite.js build` 代替。

## 七、配置 systemd 服务

```bash
cat > /etc/systemd/system/learnmate.service << 'EOF'
[Unit]
Description=LearnMate-AI Backend Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/shen_study_agent
ExecStart=/opt/shen_study_agent/venv/bin/python -m uvicorn src.backend.app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable learnmate
systemctl start learnmate
```
