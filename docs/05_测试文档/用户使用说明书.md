## 1. 系统访问

### 云端访问（推荐）

直接在浏览器中打开：

### 本地开发

```bash

pip install -r requirements.txt
uvicorn src.backend.app.main:app --reload

# 前端
cd src/frontend
npm install
npm run dev
```

## 2. 界面布局

系统采用左右布局：

- **左侧侧边栏**（220px 固定）：Logo、学生选择器、导航菜单
- **右侧内容区**：根据导航显示不同页面

### 侧边栏导航

| 导航项 | 说明 | 是否需要先生成方案 |
|--------|------|-------------------|
| 首页 | 主页，生成学习方案 | 否 |
| 讲义 | 知识讲解详情 | 是 |
| 练习题 | 交互式答题 | 是 |
| 代码案例 | 代码实操详情 | 是 |
| 思维导图 | Mermaid 渲染 | 是 |
| 学习路径 | 分阶段计划 | 是 |

## 3. 快速开始

### 第一步：选择学生

#### 新学生

1. 在主页的结构化表单中填写信息
2. 或点击预设按钮快速填入：
   - **零基础入门型**：通俗讲解，混合题型
   - **项目实战型**：实操导向，编程题
   - **考试冲刺型**：考点速览，选择题

#### 已有学生

1. 点击侧边栏学生选择器的"切换"按钮
2. 从列表中选择学生
3. 表单自动填入该学生的画像信息

### 第二步：填写结构化字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | 学生姓名 |
| 专业 | 文本 | 如"人工智能"、"计算机科学" |
| 学习目标 | 单选 | 期末提分/入门掌握/项目实战/考研准备/竞赛准备 |
| 基础水平 | 单选 | 零基础/基础一般/有编程基础/学过部分算法 |
| 薄弱知识点 | 多选 | 线性回归/逻辑回归/决策树/支持向量机/聚类/模型评估/特征工程 |
| 学习风格 | 单选 | 喜欢看讲解/喜欢做题/喜欢代码实战/喜欢图示化总结 |
| 每周投入 | 单选 | 5小时/8小时/10小时/15小时 |
| 练习偏好 | 单选 | 选择题/简答题/编程题/混合题型 |
| 补充说明 | 文本 | 其他个性化需求 |

### 第三步：生成方案

点击"生成个性化学习方案"按钮，等待约 30-60 秒。系统依次运行 6 个智能体，页面实时显示管线进度。

生成完成后：
- 结果展示在主页各面板中
- 侧边栏资源导航项解锁
- 学生信息保存到浏览器 localStorage

## 4. 功能详解

### 4.1 知识图谱

主页底部展示课程知识图谱：
- 8 个 ML 模块节点按依赖关系连线
- 薄弱知识点自动高亮为红色
- 上游依赖高亮为橙色
- 提示语："系统发现你 XX 薄弱，建议先巩固 YY 基础"

### 4.2 交互式练习题

点击侧边栏"练习题"进入答题页面。

**选择题：**
- 显示题目 + A/B/C/D 选项
- 点击选项即时显示对错
- 绿色 = 正确，红色 = 错误（显示正确答案）
- 成绩自动提交，无需手动操作

**简答题：**
- 显示题目 + 文本输入框
- 输入答案后点击"提交批改"
- AI 自动评分，显示分数、反馈、命中要点、遗漏要点
- 成绩自动提交

**题型说明：**
- 练习题类型受"练习偏好"设置影响
- 选择题偏好 → 主要生成选择题
- 简答题偏好 → 主要生成简答题
- 混合题型 → 两者混合

### 4.3 苏格拉底式追问

在智能答疑区（主页底部）：
- 输入问题后，AI 基于知识库回答
- 开启"苏格拉底模式"开关后，AI 会在回答末尾追加一个引导性问题
- 例如：问"什么是过拟合？"，AI 回答后追问"你能想出一种正则化方法来缓解它吗？"

### 4.4 学生持续跟踪

- 学生信息自动保存到浏览器
- 刷新页面后自动恢复上次的结果
- 侧边栏可随时切换不同学生
- 同一学生多次"生成"不重复创建记录
- 评估历史跨次累积

### 4.5 教师/评委看板

在主页成果看板区域：
- 查看学生总数、资源生成统计
- 查看弱项知识点分布
- 点击学生展开查看详细画像、诊断、评估历史

## 5. 查看学习效果评估

### 自动提交

练习题页面答题完成后，成绩自动提交到后端，无需手动操作。

### 查看评估历史

1. 主页评估历史面板显示所有成绩记录
2. 包含知识点、分数、反馈、趋势（上升/下降/首次）
3. 弱项知识点（平均分低于 70）会被标记

### 评估对推荐的影响

- 系统会根据评估历史调整下次推荐的重点
- 弱项知识点会被优先推荐
- 推荐依据中明确展示"因为 XX 知识点得分较低，建议重点复习"

## 6. 切换大模型

在服务器 `src/backend/app/config.py` 中修改配置：

```python
USE_LLM = True
LLM_PROVIDER = "generic"
LLM_BASE_URL = "https://api.deepseek.com"
LLM_API_KEY_V2 = "your-api-key"
LLM_MODEL = "deepseek-chat"
```

重启后端：`systemctl restart learnmate`
