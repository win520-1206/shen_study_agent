# 系统架构设计

> 更新日期：2026-06-04
> 版本：V1.0

---

## 总体架构

- 前端：Vue 3 + Vite + Element Plus + vue-router + ECharts
- 后端：FastAPI + SQLAlchemy + SQLite
- 知识库：Markdown + JSON 元数据 + 知识依赖图谱
- 编排方式：显式多智能体角色协同，支持 LLM/规则式双模式切换

### 状态管理

使用 Vue 3 `reactive` 实现简易全局状态（`store.ts`）：

- `appState.studentId` / `studentName` — 当前学生身份
- `appState.result` — 最近一次生成的完整方案
- `persistStudent()` / `clearStudent()` — localStorage 持久化工具函数
- localStorage key: `learnmate_student`

## 智能体角色

| 编号 | 智能体 | 实现类 | 职责 |
|------|--------|--------|------|
| 1 | 画像智能体 | LLMProfileAgent | 从自然语言提取 8 维学习画像 |
| 2 | 诊断智能体 | LLMDiagnosisAgent | 分析薄弱环节，推荐优先模块 |
| 3 | 资源规划智能体 | LLMContentGeneratorAgent | 决定资源难度、风格和类型 |
| 4 | 内容生成智能体 | LLMContentGeneratorAgent | 生成 5 类个性化学习资源 |
| 5 | 路径规划智能体 | (编排器内置) | 设计分阶段学习路径 |
| 6 | 审查智能体 | (ReviewAgent) | 校验内容完整性和来源可追溯性 |
| 7 | 自动批改智能体 | LLMQuizGraderAgent | 对练习题 LLM 评分并返回反馈 |

另有 `LLMQAAgent` 负责智能答疑，支持普通模式和苏格拉底追问模式。

## 数据流

### 主流程（生成个性化学习方案）

```
用户输入（结构化表单 + 补充说明）
    ↓
创建/复用学生 → persistStudent → localStorage
    ↓
┌─────────────────────────────────────────────────┐
│  1. 画像智能体：提取 8 维画像                     │
│  2. 诊断智能体：分析薄弱 + 推荐模块               │
│  3. 资源规划：决定 5 类资源参数                    │
│  4. 内容生成：生成讲义/题目/案例/导图/路径         │
│  5. 路径规划：设计 3 阶段学习路线                  │
│  6. 审查智能体：校验完整性                         │
└─────────────────────────────────────────────────┘
    ↓
返回 ProfileBuildResponse → 前端渲染
    ↓
侧边栏资源导航项解锁 → 点击进入各资源详情页
```

### 练习答题流程

```
用户进入练习题页 → QuizPanel 解析 markdown 为结构化题目
    ↓
选择题：点击选项 → 前端即时判对错 → 自动 POST /assessment/submit
简答题：输入答案 → POST /quiz/grade → LLM 评分 → 自动 POST /assessment/submit
    ↓
评估历史自动更新 → 影响下次生成的推荐策略
```

### 智能答疑流程

```
用户在 QAPanel 输入问题
    ↓
POST /api/v1/chat/qa（socratic: true/false）
    ↓
LLMQAAgent 基于知识库回答
    ↓
苏格拉底模式下追加引导性问题
```

## 知识图谱模块

- 数据文件：`data/course_kb/knowledge_graph.json`
- 包含 8 个 ML 模块节点（机器学习概述、数据预处理、线性回归、逻辑回归、决策树、支持向量机、聚类、模型评估）
- 节点间有依赖关系边（如：梯度下降 → 线性回归 → 逻辑回归）
- 前端使用 ECharts 力导向图渲染
- 诊断出薄弱点时自动高亮相关节点及上游依赖

## LLM 服务层

```
LLMService (基类)
├── chat(system_prompt, user_prompt) -> str
└── chat_json(system_prompt, user_prompt) -> dict

MockLLMService          # 开发测试用，不调用真实 API
GenericLLMService       # 通用 OpenAI 兼容接口（DeepSeek 等）
```

通过 `config.py` 中的 `USE_LLM` 开关控制使用 LLM 模式还是规则模式。

## API 端点总览

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/students` | POST | 创建学生 |

| `/api/v1/student/{id}/assessments` | GET | 评估历史与弱项分析 |
| `/api/v1/student/{id}/dashboard` | GET | 学生完整学习状态 |
| `/api/v1/overview/summary` | GET | 教师/评委成果看板 |
| `/api/v1/kb/graph` | GET | 知识依赖图谱 JSON |
| `/api/v1/kb/modules` | GET | 课程模块索引 |
| `/api/v1/kb/questions` | GET | 题库 |
| `/api/v1/kb/coding-cases` | GET | 代码案例 |
