## 1. 背景与目标

### 1.1 当前状态

LearnMate-AI 系统的 **全部 5 个 LLM 智能体** 已接入大模型，通过 DeepSeek API 提供服务。所有智能体均支持 Mock 模式（开发测试）和真实 LLM 模式（一行配置切换）。

已接入 LLM 的智能体：

| 智能体 | 实现类 | 接入状态 |
|--------|--------|----------|
| 画像智能体 | LLMProfileAgent | 已接入 |
| 诊断智能体 | LLMDiagnosisAgent | 已接入 |
| 内容生成智能体 | LLMContentGeneratorAgent | 已接入 |
| 答疑智能体 | LLMQAAgent | 已接入 |
| 自动批改智能体 | LLMQuizGraderAgent | 已接入 |

### 1.2 设计优势

1. **一行配置切换**：通过 `USE_LLM` 开关在规则式和大模型模式间切换
2. **前端零改动**：数据结构完全兼容，前端无需任何修改
3. **统一接口**：所有智能体返回 `AgentResult`，编排器无感知

## 2. 架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                   编排器 Orchestrator             │
│                                                   │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐   │
│  │ 画像智能体 │  │ 诊断智能体│  │ 内容生成智能体│   │
│  │ [LLM]    │  │ [LLM]    │  │   [LLM]      │   │
│  └──────────┘  └──────────┘  └──────────────┘   │
│                                                   │
│  ┌──────────┐  ┌──────────────┐  ┌──────────┐   │
│  │ 答疑智能体│  │ 自动批改智能体│  │ 审查智能体│   │
│  │  [LLM]   │  │   [LLM]      │  │  [规则式] │   │
│  └──────────┘  └──────────────┘  └──────────┘   │
│                                                   │
│  ┌─────────────────────────────────────────┐     │
│  │         LLM Service Layer               │     │
│  │  ┌──────────┐  ┌──────────────────────┐ │     │
│  │  │MockLLM   │  │GenericLLM(DeepSeek) │ │     │
│  │  └──────────┘  └──────────────────────┘ │     │
│  └─────────────────────────────────────────┘     │
└─────────────────────────────────────────────────┘
```

### 2.2 配置驱动切换

在 `config.py` 中通过配置项控制：

```python
# 设为 True 启用大模型智能体，设为 False 使用规则式智能体
USE_LLM = True

# LLM 服务提供者: "mock" / "generic"
LLM_PROVIDER = "generic"

# 通用 LLM 配置（兼容 OpenAI 接口）
LLM_BASE_URL = "https://api.deepseek.com"
LLM_API_KEY_V2 = "sk-你的密钥"
LLM_MODEL = "deepseek-chat"
```

### 2.3 LLM 服务层抽象

```
LLMService (基类)
├── chat(system_prompt, user_prompt) -> str
└── chat_json(system_prompt, user_prompt) -> dict

MockLLMService          # 开发测试用，返回预设响应
GenericLLMService       # 通用 OpenAI 兼容接口（DeepSeek 等）
```

`chat_json` 方法自动处理 LLM 返回的 JSON 解析，包含容错逻辑（提取 JSON 块、处理 markdown 包裹等）。

## 3. 各智能体 Prompt 设计

### 3.1 LLMProfileAgent

- 输入：学生自然语言描述
- 输出：8 维画像 JSON
- 特点：理解否定句和复杂意图（如"我不想一开始就上很多代码"正确识别为不喜欢代码）

### 3.2 LLMDiagnosisAgent

- 输入：画像 + 知识库检索结果
- 输出：当前阶段、优先模块、推荐策略、风险点
- 特点：结合知识模块结构做分析，推荐更具逻辑性

### 3.3 LLMContentGeneratorAgent

- 输入：画像 + 诊断 + 知识库内容
- 输出：5 类资源（讲义、练习题、代码案例、思维导图、学习路径）
- 特点：
  - 练习题格式受 `exercise_preference` 控制
  - 选择题使用 `A./B./C./D.` 格式，参考答案标注"正确答案是X"
  - 内容基于知识库生成，减少幻觉

### 3.4 LLMQAAgent

- 输入：学生问题 + 知识库上下文
- 输出：回答 + 来源引用
- 双模式：
  - **普通模式**：直接回答，控制 300 字以内
  - **苏格拉底模式**：回答 + 引导性追问，回答主体 200 字以内，末尾追加 1 个思考问题

### 3.5 LLMQuizGraderAgent

- 输入：题目 + 参考答案 + 学生答案 + 知识点
- 输出：`{"score": 0-100, "feedback": "...", "key_points_hit": [...], "key_points_missed": [...]}`
- 特点：
  - 语义匹配而非精确匹配
  - 返回命中/遗漏的具体要点
  - LLM 失败时 fallback 到关键词匹配

## 4. Prompt 工程实践

### 4.1 知识库作为上下文（RAG）

所有内容生成和答疑都基于课程知识库，不凭空生成：

```
用户输入 → 画像抽取 → 知识库检索 → 检索结果 + 画像 → LLM Prompt → 生成内容
```

### 4.2 JSON 输出约束

所有需要结构化输出的智能体都使用 `chat_json()` 方法：
- Prompt 明确要求输出 JSON 格式
- 自动提取 LLM 返回中的 JSON 块
- 包含多层容错（正则提取、markdown 清理等）

### 4.3 降级策略

当 LLM 调用失败时：
- 画像智能体：fallback 到关键词匹配
- 内容生成：fallback 到模板生成
- 自动批改：fallback 到关键词匹配评分
- 答疑：返回"暂时无法回答"提示
