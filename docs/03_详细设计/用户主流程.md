# 用户主流程

## 1. 进入系统

学生输入姓名、专业和当前学习诉求，例如：

- 我是人工智能专业学生
- 机器学习零基础
- 想通过项目实战学会线性回归和模型评估
- 希望多给代码案例

## 2. 系统构建学生画像

画像智能体从自然语言中抽取核心信息，形成 8 维画像：

1. 专业背景
2. 学习目标
3. 先修基础
4. 薄弱知识点
5. 学习风格
6. 每周可投入时间
7. 练习偏好
8. 目标结果

## 3. 系统进行课程诊断

诊断智能体结合画像和知识库检索结果，判断：

- 学生当前处于哪个学习阶段
- 优先学习哪些模块
- 当前最大风险点是什么
- 最值得聚焦的知识单元是什么

## 4. 系统规划资源集合

资源规划智能体决定本轮输出哪些类型的学习资源，并确定难度和表达风格。

第一版固定输出：

1. 个性化讲解文档
2. 练习题
3. 代码实操案例
4. 思维导图
5. 学习任务清单

## 5. 系统生成资源

内容生成智能体根据知识库内容与学生画像，生成与当前学习目标匹配的资源内容。  
每条资源都要带来源引用，保证内容可追溯。

## 6. 系统生成学习路径

路径规划智能体把资源组织为 3 个阶段：

1. 打基础
2. 做练习
3. 做代码

每个阶段都包含：

- 学习目标
- 推荐资源
- 实践任务

## 7. 系统执行结果审查

审查智能体检查：

- 资源数量是否足够
- 是否都标注了来源
- 是否存在结构缺失

## 8. 学生进行练习与反馈

学生提交某个知识点的练习成绩后，系统根据得分给出反馈，并调整下一步推荐。

## 9. Dashboard 汇总展示

系统最终在 Dashboard 中集中展示：

- 学生画像
- 最近一次诊断
- 当前学习路径
- 本轮资源卡片
- 历史测验反馈
- 多智能体轨迹
