## 今日完成内容

### 1. 后端 LLM 流式支持

- `LLMService` 基类新增 `chat_stream()` 方法（默认 fallback 到 `chat()`）
- `GenericLLMService` 实现真正的 SSE 流式调用：
  - 使用 `httpx.Client.stream()` 调用 OpenAI 兼容 API 的 `stream: true` 模式
  - 逐行解析 `data: {...}` 格式的 SSE 响应
  - 提取 `choices[0].delta.content` 逐 token 返回
  - 错误时 fallback 到 Mock

### 2. 后端 SSE 流式端点

- 新增 `src/backend/app/routers/stream.py`
- 端点：`POST /api/v1/chat/profile-build/stream`
- 流式事件顺序：profile -> diagnosis -> traces -> resource(逐个) -> plan -> done
- 使用 FastAPI `StreamingResponse(media_type="text/event-stream")`
- 注册到 `main.py`

### 3. 前端流式支持

- `useApi.ts` 新增 `buildProfileStream()` 函数
  - 使用 `fetch()` + `ReadableStream` 接收 SSE
  - 解析 event + data 行格式
  - 通过回调函数逐事件更新状态
- `App.vue` 新增流式输出模式开关
  - 复选框切换标准/流式模式
  - 流式模式下资源卡片逐个出现
  - 流式模式下先显示空壳，逐步填充数据
