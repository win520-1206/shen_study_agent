## 四、关键问题与解决过程

### 1. 问题：服务无法启动

启动时报错：

```text
ImportError: cannot import name 'UTC' from 'datetime'
```

原因分析：

1. 服务器默认使用 Python 3.10。
2. 项目代码依赖较新的 `datetime.UTC` 写法。
3. 因此旧版本 Python 无法兼容当前代码。

解决措施：

1. 在服务器安装 Python 3.13。
2. 删除旧虚拟环境。
3. 使用 Python 3.13 重新创建 `venv`。
4. 重新安装依赖并启动服务。

结果：

服务成功启动，浏览器可以正常访问。

### 2. 问题：需要长期稳定运行

原因分析：

如果只用手动命令运行 `uvicorn`，关闭 SSH 终端后服务会中断，不适合长期演示和公网使用。

解决措施：

1. 配置 `systemd` 服务文件。
2. 将项目注册为 `learnmate` 服务。
3. 设置后续通过 `systemctl restart learnmate` 统一管理。

结果：

项目具备后台运行、重启和开机自启能力。

## 八、经验总结

1. 云服务器部署时，首先要确认 Python 版本是否与项目代码兼容。
2. 本地能跑通不代表服务器能直接运行，环境差异常常是首要问题。
3. 手动运行适合排错，`systemd` 更适合正式运行。
4. 部署完成后必须同步整理文档，否则后续维护成本会迅速上升。
5. 含有 API Key 的真实配置文件不能上传到 GitHub。
